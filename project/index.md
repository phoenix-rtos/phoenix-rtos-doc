# Project repository

## Synopsis

After reading this chapter, you will know:

- How the Phoenix-RTOS project is organized as a collection of Git submodules
- The purpose of each repository in the project
- How the build entry point (`build.project`) and output directories work
- How to use Docker for building and development

The main repository of Phoenix-RTOS is the
[phoenix-rtos-project](https://github.com/phoenix-rtos/phoenix-rtos-project.git).

## Build Entry Point

The project root contains no Makefiles. Building is done through the shell-based build system:

```shell
TARGET=ia32-generic-qemu ./phoenix-rtos-build/build.sh all
```

The build script sources `build.project` from the project root, which defines board-specific build functions. Each
submodule has its own Makefiles, but they are invoked by the build system  -  not called directly.

## Project Structure

The project consists of the following directories and files:

### Submodule Repositories

- [libphoenix](https://github.com/phoenix-rtos/libphoenix.git)

    Standard C library. Written from scratch for Phoenix-RTOS.

- [phoenix-rtos-build](https://github.com/phoenix-rtos/phoenix-rtos-build.git)

    Building scripts, makefile templates, rules, flags definitions, target selection and
    toolchain.
- [phoenix-rtos-corelibs](https://github.com/phoenix-rtos/phoenix-rtos-corelibs.git)

    Libraries for use in user space.

- [phoenix-rtos-devices](https://github.com/phoenix-rtos/phoenix-rtos-devices.git)

    Hardware drivers.

- [phoenix-rtos-doc](https://github.com/phoenix-rtos/phoenix-rtos-doc.git)

    Documentation.

- [phoenix-rtos-filesystems](https://github.com/phoenix-rtos/phoenix-rtos-filesystems.git)

    Filesystem drivers.

- [phoenix-rtos-hostutils](https://github.com/phoenix-rtos/phoenix-rtos-hostutils.git)

    Utilities for development PC (e.g. a tool for transferring system binary image to the
    target).
- [phoenix-rtos-kernel](https://github.com/phoenix-rtos/phoenix-rtos-kernel.git)

    Microkernel repository.

- [phoenix-rtos-lwip](https://github.com/phoenix-rtos/phoenix-rtos-lwip.git)

    LwIP network stack.

- [phoenix-rtos-ports](https://github.com/phoenix-rtos/phoenix-rtos-ports.git)

    Linux (and potentially other OSes) applications ported to Phoenix-RTOS.

- [phoenix-rtos-posixsrv](https://github.com/phoenix-rtos/phoenix-rtos-posixsrv.git)

    POSIX server; user space server that is providing additional POSIX features not
    provided by the kernel itself (e.g.
    pipes).
- [phoenix-rtos-tests](https://github.com/phoenix-rtos/phoenix-rtos-tests.git)

    Tests based on our own framework.

- [phoenix-rtos-usb](https://github.com/phoenix-rtos/phoenix-rtos-usb.git)

    USB stack (both host and device).

- [phoenix-rtos-utils](https://github.com/phoenix-rtos/phoenix-rtos-utils.git)

    System utilities (e.g. native shell psh).

- [plo](https://github.com/phoenix-rtos/plo.git)

    Phoenix-RTOS bootloader.

### Project-Level Directories and Files

| Path | Purpose |
|------|---------|
| `_projects/` | Per-target project configuration  -  each subdirectory contains a `build.project` with board-specific build functions and optionally `ports.yaml` for port selection |
| `_targets/` | Per-architecture build rules (`build.common` plus architecture subdirectories) |
| `_fs/` | Rootfs templates and overlays  -  per-target filesystem content merged into the final image |
| `_boot/` | Output: final bootable images (kernel + applications + rootfs) |
| `_build/` | Output: intermediate build artifacts (object files, libraries, sysroot) |
| `_user/` | Example user applications (hello, hellocpp, msgmark, rotrectangle, serverdemo, voxeldemo) with their own Makefile |
| `scripts/` | Launch scripts for QEMU/simulator targets (e.g., `ia32-generic-qemu.sh`, `riscv64-generic-qemu.sh`) |
| `mtd-utils/` | Tool for creating JFFS2 filesystem partitions |
| `build.project` | Project-level build configuration  -  sourced by `phoenix-rtos-build/build.sh` |
| `busybox_config` | Default BusyBox configuration for targets that include BusyBox as a port |
| `compile_commands.json` | IDE/LSP compilation database  -  auto-generated by `bear` when available |
| `docker-build.sh` | Docker-based isolated build (macOS tmpfs optimization, `.docker_env` variable passthrough) |
| `docker-devel.sh` | Docker-based interactive development with USB passthrough for hardware targets |

### Submodule Management

All repositories listed above are Git submodules. To clone the full project with all submodules:

```shell
git clone --recursive https://github.com/phoenix-rtos/phoenix-rtos-project.git
```
