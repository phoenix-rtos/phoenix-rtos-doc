# Loader (PLO) Documentation — Undocumented Areas

## 1. Undocumented CLI Commands

Source directory: `plo/cmds/`. 36 command files total. Beyond the ~20 documented commands, the following are undocumented:

### Bootloader-Specific
- **bootcm4** (`bootcm4.c`): Boot Cortex-M4 core on dual-core i.MX RT
- **bootrom** (`bootrom.c`): Access ROM bootloader
- **blob** (`blob.c`): Load embedded data blob
- **bitstream** (`bitstream.c`): Load FPGA bitstream
- **call** (`call.c`): Call script from file

### Diagnostic/Administration
- **devices** (`devices.c`): Enumerate available devices
- **erase** (`erase.c`): Erase flash memory regions
- **mem** (`mem.c`): Read/write memory directly
- **watchdog** (`watchdog.c`): Configure hardware watchdog
- **test-dev** (`test-dev.c`): Test device functionality
- **test-ddr** (`test-ddr.c`): Test DDR memory
- **reboot** (`reboot.c`): System reboot
- **mpu** (`mpu.c`): MPU configuration

### Format/Configuration Tools
- **jffs2** (`jffs2.c`): Write JFFS2 cleanmarkers on flash
- **ptable** (`ptable.c`): Parse and display partition table
- **kernelimg** (`kernelimg.c`): Load XIP (Execute-in-Place) kernel image

### Platform-Specific (IA32)
- **vbe** (`vbe.c`): VGA BIOS Extension modesetting
- **lspci** (`lspci.c`): Enumerate PCI bus devices

### Hardware Programming
- **otp** (`otp.c`): Program OTP (One-Time Programmable) fuses
- **bridge** (`bridge.c`): Serial device multiplexing

### Flow Control
- **stop** (`stop.c`): Halt script execution

## 3. RISC-V SBI Subsystem

Complete module at `plo/riscv-sbi/` for RISC-V Supervisor Binary Interface. Contains:
- `entry.c` — SBI entry point
- `core/` — SBI core implementation
- `devices/` — SBI device drivers
- `platform/` — Platform-specific code
- `include/` — Headers
- `ld/` — Linker scripts

Absent from architecture documentation.

## 5. Command Registration Architecture

Uses custom linker section `.commands` with `__cmd_start`/`__cmd_end` anchors plus `__attribute__((constructor))`. Commands self-register at load time. This is not just simple registration as docs imply — it's a linker-section-based plugin system.

## 6. PLO Script System

- Embedded script support via `.data` section patching
- Script sourcing from YAML (`preinit.plo.yaml`, `user.plo.yaml`)
- Generated by `b_mkscript_user()` in `_targets/build.common`
- Split into target baseline + project override
- `call` command (`plo/cmds/call.c`) can execute scripts from PHFS files

## 7. PHFS Protocol Details

Source: `plo/phfs/phfs.c`.
- "raw" protocol for direct device access
- "phoenixd" protocol for host communication
- Hardcoded limits: `SIZE_PHFS_ALIASES = 32` file aliases (line 22)
- Protocol details and message format not documented

## 8. Command Line Limits

Source: `plo/cmds/cmd.h` and `plo/cmds/cmd.c`.
- `SIZE_CMD_ARG_LINE = 256` bytes argument line (cmd.h line 23)
- `SIZE_CMD_ARGV = 10 + 1` arguments maximum per command (cmd.h line 28)
- `SIZE_HIST = 8` history entries (cmd.c line 23)
- These limits may affect complex scripting scenarios
