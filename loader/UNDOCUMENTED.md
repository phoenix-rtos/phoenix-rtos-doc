# Loader (PLO) Documentation — Undocumented Areas

## 1. Undocumented CLI Commands

### Bootloader-Specific
- **bootcm4**: Boot Cortex-M4 core on dual-core i.MX RT
- **bootrom**: Access ROM bootloader
- **blob**: Load embedded data blob

### Diagnostic/Administration
- **devices**: Enumerate available devices
- **erase**: Erase flash memory regions
- **mem**: Read/write memory directly
- **watchdog**: Configure hardware watchdog
- **test-dev**: Test device functionality
- **reboot**: System reboot

### Format/Configuration Tools
- **jffs2**: Write JFFS2 cleanmarkers on flash
- **ptable**: Parse and display partition table
- **kernelimg**: Load XIP (Execute-in-Place) kernel image

### Platform-Specific (IA32)
- **vbe**: VGA BIOS Extension modesetting
- **lspci**: Enumerate PCI bus devices

### Hardware Programming
- **otp**: Program OTP (One-Time Programmable) fuses on i.MX RT and STM32N6
- **bridge**: Serial device multiplexing (2024)

### Flow Control
- **stop**: Halt script execution (2024)

## 5. Command Registration Architecture

Uses custom linker section `.commands` with `__cmd_start`/`__cmd_end` anchors plus constructor attributes. Not just simple registration functions as docs imply.

## 6. PLO Script System

- Embedded script support via `.data` section patching
- Script sourcing from YAML (preinit.plo.yaml, user.plo.yaml)
- Generated by `b_mkscript_user()` in `_targets/build.common`
- Split into target baseline + project override

## 7. PHFS Protocol Details

- "raw" protocol for direct device access
- "phoenixd" protocol for host communication
- Hardcoded limits: 8 device handlers, 32 file aliases
- Protocol details and message format not documented

## 8. Command Line Limits

- 256-byte argument line
- 10 arguments maximum per command
- 8 history entries
- These limits may affect complex scripting scenarios
